# 校园互助学习平台 - 团队分工说明

## 技术栈
- 后端：Java + Spring Boot 2.7 + MyBatis
- 前端：Vue 3 + Element Plus
- 数据库：MySQL
- 接口文档：Swagger（启动后端后访问 http://localhost:8080/swagger-ui/index.html）

---

## 👥 新版分工（6大角色）

| 角色 | 人数 | 职责 |
|---|---|---|
| **前端开发** | 1人 | 所有页面开发 |
| **后端开发** | 2人 | 协作完成所有后端接口 |

## 🔧 启动方式

### 1. 数据库
安装 MySQL 8.0+，root 密码设为 `123456`，执行 `database/schema.sql`。

### 2. 后端
用 IDEA 打开 `backend` 文件夹，等 Maven 依赖下载完，运行 `CampusQaApplication.java`。
确认启动：http://localhost:8080/swagger-ui/index.html

### 3. 前端
```bash
cd frontend
npm install
npm run dev
```
访问 http://localhost:5173

### ☕ 后端开发（2人协作）

**所有后端文件都在 `backend/src/main/java/com/campusqa/` 下。两人商量好各自负责哪些模块即可。**

| 模块 | Controller | Service | Mapper | 涉及表 |
|---|---|---|---|---|
| 用户模块 | `UserController.java` | `UserServiceImpl.java` | `UserMapper.java`、`FollowMapper.java`、`FavoriteMapper.java` | user, follow, favorite |
| 后台管理 | `AdminController.java` | `AdminServiceImpl.java` | `AdminMapper.java`、`CourseMapper.java` | admin, course |
| 问答模块 | `QuestionController.java` | `QuestionServiceImpl.java` | `QuestionMapper.java`、`AnswerMapper.java`、`CommentMapper.java`、`QuestionLikeMapper.java` | question, answer, comment, question_like |
| 资源模块 | `ResourceController.java` | `ResourceServiceImpl.java` | `ResourceMapper.java` | resource |

**建议分工：**
- 后端①：用户模块 + 后台管理（约18个接口）
- 后端②：问答模块 + 资源模块（约13个接口）

**接口文档：** http://localhost:8080/swagger-ui/index.html（后端启动后可查看）

## ⚠️ 注意事项

1. **每天开工前** `git pull`（拉取别人更新）
2. **每天收工后** `git add .` → `git commit -m "做了什么"` → `git push`
3. 遇到冲突别慌，在群里问
4. 接口对不上先看 Swagger，再问后端
5. 数据库密码不是 123456 就改 `application.yml`
